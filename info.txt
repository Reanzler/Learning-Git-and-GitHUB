        #Команды в Git системе
git status - проверка статуса
git add <name of file> - добавление файла в ветку
git add . - добавление нескольких файлов в ветку
git branch - просмотр текущих веток
git branch <new name for file> - создание новой ветки
git checkot <name of file> - переход на нужную ветку
git branch -D <name of file> - удаление нужной ветки
git commit -m "<nay name for currently connit >" - коммит изменений в репозитории
git push - заливаем на GitHUB для обновления удаленного репозитория


#Информативно*
git commit - редактирует репозиторий, заносит туда изменения. git push - обновляет ссылки на удалённый репозиторий. 
Простыми словами, когда вы "коммитите" изменения, вы делаете их на своём локальном репозитории, при этом удалённый 
репозиторий (тот что непосредственно находится на Git) остаётся без изменений


Команда git add — это первая команда в цепочке операций, предписывающей Git «сохранить» снимок текущего состояния проекта 
в истории коммитов. Когда git add используется как отдельная команда, она переносит ожидающие изменения из рабочего каталога 
в раздел проиндексированных файлов.
